## PostgreSQL
Это объектно-реляционная СУБД.

### Особенности

* <u>Полнотекстовый поиск</u> встроен, возможен без использования внешних систем. Для него используется типы данных `tsvector` и `tsquery`, индекс `GIN`, а также опреатор `@@` (для проверки совпадения вектора и запроса)
* <u>Нечёткий поиск</u> возможен с расширением `pg_trgm`, которое строит триграммы. Функция `similarity` возвращает совпадение от 0 до 1. Индекс GIN. Оператор `%` (возвратит true если схожесть строк выше 0,3 по умолчанию)

### Преимущества

- встроенный полнотекстовый поиск
- поддержка русского языка
- поиск по словоформам
- обработка опечаток в `pg_trgm`

### Минусы

- хуже работает на огромных данных
- меньше возможностей анализа текста

## ClickHouse

Это колоночная аналитическая СУБД, ориентированная на обработку больших объёмов данных.

### Особенности
* <u>Полнотекстовый поиск</u> возможен при использовании инвертированных индексов `Text Index`. Используются в связке с этим индексом функции `hasToken` - ищет точное слово, `hasPhrase` - ищет целиком фразу

* <u>Нечёткий поиск</u> возможен при использовании ngram-индексов (`ngrambf_v1`) и функции расстояния ( `editDistance` - расстояние Левенштейна, `ngramDistance` - аналог функции `similarity` из PostgreSQL).  

### Преимущества

- высокая скорость поиска по огромным объёмам данных
- хорошая масштабируемость
- встроенные текстовые индексы

### Минусы
- меньше инструментов для обработки естественного языка
- ориентирован на аналитику, а не на поиск
-  меньше развит нечёткий поиск по сравнению с PostgreSQL и Elasticsearch

## Elasticsearch

Это специализированная распределённая поисковая система, построенная на базе Apache Lucene

### Особенности
* <u>Полнотекстовый поиск</u> осуществляется при помощи инвертированного индекса Lucene. Поисковые функции `match` - базовый поиск по одному полю, `multi_match` - поиск по нескольким полям, `match_phrase` - поиск точной фразы, `query_string` - с поддержкой логических операторов. 

* <u>Нечёткий поиск</u> основан на расстоянии Левенштейна. В обычный запрос `match` (`multi_match`) добавляется поле `fuzziness` (указывает, сколько максимальных изменений разрешено сделать в слове). Возможен режим `fuzziness: AUTO`. Для оценки релевантности результатов используется алгоритм BM25.

### Преимущества

- самая развитая поисковая система
- удобное масштабирование
- большое количество анализаторов
- высокая производительность на больших индексах
- поддержка множества языков

### Минусы

- синхронизация данных с основной БД
- повышенное потребление памяти
- более сложная настройка 
- необходимо разворачивать отдельный кластер
